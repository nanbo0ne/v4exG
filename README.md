# ✍️ 长篇小说写作助手 V4.3.4 exG

一个面向长篇小说创作的单文件 HTML 工具。它把项目设定、大纲、章节正文、长期记忆、创作规则、参考风格、审稿修改和导出备份集中在一个浏览器页面里，适合用于从一句话构思逐步扩展到大纲、分章正文、润色和全书维护。

🌐 在线访问：[nanbo0ne.github.io/v4exG](https://nanbo0ne.github.io/v4exG/)

📦 项目仓库：[github.com/nanbo0ne/v4exG](https://github.com/nanbo0ne/v4exG)

## ✨ 核心特点

- 🧩 **单文件运行**：下载 HTML 后即可在浏览器中打开使用，不需要安装依赖或部署后端。
- 💾 **本地存储**：项目数据保存在浏览器 IndexedDB 中，支持 JSON 备份与恢复。
- 🔌 **多 API 配置**：可填写 OpenAI 兼容接口地址、API Key 和模型名，适配不同模型服务。
- 🧱 **大纲到正文流程**：支持生成全书大纲、分章生成、续写、写结尾、补写和去 AI 味处理。
- 🎨 **创作规则 / 参考风格**：支持叙事视角、基调氛围、禁止事项、写作风格约束、引用手法、大师洞察和写作卡片。
- 🧪 **参考文本分析**：可从参考文本中生成写作卡片、引用手法和大师洞察，生成正文时只注入提炼后的方法，不直接发送参考原文。
- 🧠 **长期记忆与章节承接**：自动维护角色状态、重要事件、世界观、伏笔、章节摘要卡和上一章桥接信息。
- 🧐 **审稿模式**：支持一致性检查、专家意见修改、按章勾选建议后改写，以及一键生成后的自动审稿流程。
- ⚡ **缓存友好上下文**：V4.3.4 优化了全文模式和摘要模式的请求排列，尽量让已完成前文形成稳定缓存前缀。

## 🚀 V4.3.4 新增优化

### 📚 全文参考模式

全文模式不再把前文简单拼进一个大 prompt，而是把已完成章节按对话式上下文放在当前任务前面：

```text
system: 固定写作身份
user: 第 1 章信息
assistant: 第 1 章定稿正文
user: 第 2 章信息
assistant: 第 2 章定稿正文
user: 长期记忆、章节承接、本章要求
user: 当前章节生成任务
```

这样前面已经定稿的章节更容易形成稳定前缀，适合支持 prompt cache 的大上下文模型。

### 📋 摘要参考模式

摘要模式会使用“章节摘要卡账本”。每章定稿后，工具会为该章生成一张摘要卡，记录：

- 本章实际剧情结果
- 关键事件
- 人物变化
- 新增或回收的伏笔
- 重要地点、道具和规则
- 章末状态
- 检索关键词

生成后续章节时，摘要卡会按章节顺序作为前文摘要上下文发送。若摘要卡缺失或过期，会使用本地兜底摘要，避免突然批量消耗 API。

## 🛠️ 基本使用

1. 🌐 打开网页或下载 HTML 文件。
2. 🔑 在“设置”中填写 API 地址、API Key 和模型名。
3. 📖 在“项目信息”中填写标题、类型、世界观、人物设定、章节数和每章字数。
4. 🎛️ 按需要填写“创作规则 / 参考风格”。
5. 🧭 点击“生成大纲”。
6. ✍️ 进入“写作”页，选择摘要或全文参考模式，逐章生成正文。
7. 🧹 使用续写、写结尾、去 AI、补充写作和审稿功能进行修订。
8. 📤 通过导出功能保存 TXT、Markdown、HTML 或 JSON 备份。

## 🎨 创作规则与写作卡片

创作规则用于约束当前项目的整体写法，包括：

- 叙事视角
- 基调氛围
- 禁止事项
- 写作风格约束
- 引用手法
- 大师洞察

写作卡片由参考文本提炼而来，用于总结可迁移的写作方法。参考文本默认只保存在浏览器本地，只有在点击生成写作卡片、引用手法或大师洞察时才会发送给 AI。

## 🧠 长期记忆

工具会在章节完成后自动更新长期记忆，用于维护长篇连贯性：

- 角色状态
- 地点与势力
- 重要事件
- 世界观设定
- 未解与已回收伏笔
- 章节摘要卡

V4.3.4 的摘要模式会优先使用章节摘要卡作为前文账本，减少反复拼接正文片段带来的混乱。

## 🧐 审稿与修改

审稿功能包含两类模式：

- **一致性检查**：找出人物、时间线、世界观、伏笔等前后冲突，并可生成修复指令。
- **专家意见修改**：为当前章节或全书提出节奏、冲突、转折、人物动机和爽点方面的优化建议。

一键生成中也可以启用审稿流程。专家建议不会默认强行覆盖正文，用户可以浏览建议后选择要应用的修改。

## 🔐 数据与隐私

- 💾 项目数据保存在浏览器本地 IndexedDB。
- 🔑 API Key 会保存在本地浏览器数据中，请不要把包含密钥的备份文件上传到公开仓库。
- 📚 参考文本不会自动发送给 AI，只有用户主动点击分析按钮时才会发送。
- 🧳 JSON 项目备份会包含项目设定、大纲、章节、创作规则、写作卡片、审稿报告和长期记忆。

## 📤 导出格式

支持导出：

- TXT 正文
- Markdown 正文
- HTML 正文
- JSON 项目备份
- 写作卡片 JSON

正文导出不会包含参考文本和写作卡片内容；JSON 项目备份用于恢复工程状态。

## 💡 使用建议

- 📋 章节较少或模型较弱时，优先使用摘要模式。
- 📚 需要更强前文连贯性且模型支持长上下文时，使用全文模式。
- 🚀 一键生成长篇时，建议开启章节承接、长期记忆和审稿。
- ⚡ 若使用支持 prompt cache 的云端模型，V4.3.4 的全文/摘要上下文排列会更容易复用前文缓存。
- 🖥️ 本地模型通常没有价格缓存优势，建议使用摘要模式减少上下文长度。
