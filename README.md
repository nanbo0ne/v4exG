# ✍️ 长篇小说写作助手 V4.3.8-exG-mobile

一个面向长篇小说创作的单文件 HTML 工具。它把项目设定、大纲、分章正文、长期记忆、创作规则、参考风格、审稿修改、导出备份和移动端写作体验集中在一个浏览器页面里，适合从一句话灵感一路推进到几十章、几百章的持续创作。

🌐 在线访问：[nanbo0ne.github.io/v4exG](https://nanbo0ne.github.io/v4exG/)

📦 项目仓库：[github.com/nanbo0ne/v4exG](https://github.com/nanbo0ne/v4exG)

```text

```

## ✨ 核心功能

- 🧩 **单文件运行**：下载 HTML 后即可在浏览器中打开，不需要安装依赖、不需要后端服务。
- 💾 **本地 IndexedDB 存储**：项目、章节、聊天、参考文本、写作卡片、记忆等都保存在浏览器本地。
- 🔌 **OpenAI 兼容 API**：支持填写 Base URL、API Key、模型名，可连接云端模型或本地兼容接口。
- 🧠 **任务模型分工**：可为全局思维、正文创作、去 AI/改写/专家建议、批注记忆、JSON 修复分别选择实际使用模型。
- 🧱 **从大纲到正文**：支持生成全书大纲、分章正文、续写、写结尾、补充写作、去 AI 味和一键生成。
- 🎨 **创作规则 / 参考风格**：支持叙事视角、基调氛围、禁止事项、写作风格约束、引用手法、大师洞察。
- 📚 **参考文本与写作卡片**：可从参考文本中提炼写作卡片、引用手法和大师洞察，正文生成时只注入提炼结果。
- 🧭 **摘要 / 全文前文模式**：可按项目规模选择摘要账本或全文账本，提高长篇连贯性。
- 📝 **本章批注 / 全书要求**：支持为当前章节和全书生成写作注意事项，并在写作时作为要求注入。
- 🧐 **审稿系统**：支持一致性检查、专家意见修改、按章勾选建议、专家意见应用到全文。
- 🧯 **安全改写**：手动去 AI、修改本章等操作支持改写前后对比，确认后再覆盖正文。
- 🕘 **章节历史**：关键覆盖操作前自动保存快照，支持查看和恢复当前章历史版本。
- 📤 **多格式导出**：支持 TXT、Markdown、HTML、JSON 项目备份和写作卡片 JSON 导入导出。
- 📱 **移动端 App 式界面**：手机和平板窄屏自动进入独立移动端布局，桌面端保持原有布局。

## 🚀 快速开始

1. 🌐 打开在线页面，或下载当前 HTML 文件后直接用浏览器打开。
2. 🔑 在“设置”里填写 API 地址、API Key 和模型。
3. 📖 在“项目信息”中填写标题、类型、世界观、人物设定、章节数和每章字数。
4. 🎛️ 按需要填写“创作规则 / 参考风格”，也可以导入参考文本生成写作卡片。
5. 🧭 点击“生成大纲”，确认章节规划。
6. ✍️ 进入“写作”页，选择摘要或全文参考模式，逐章生成正文。
7. 🧹 使用续写、写结尾、去 AI、修改本章和审稿功能进行修订。
8. 📤 通过导出功能保存正文或 JSON 项目备份。

## 🧠 长篇连贯性设计

工具目前提供两种前文参考方式：

- 📋 **摘要模式**：每章定稿后生成章节摘要卡，后续章节按顺序读取摘要账本。适合模型较便宜、上下文较短或章节很多的项目。
- 📚 **全文模式**：把已完成章节作为稳定的对话式前文账本发送。适合长上下文模型，尤其适合支持 prompt cache 的接口。

长期记忆会记录角色状态、重要事件、地点势力、世界观设定、伏笔与章节承接信息。它不是替代全文，而是帮助模型在长篇写作中抓住“现在发生到哪一步”。

## 🎨 创作规则与参考风格

创作规则用于约束整本书的写法，字段为空时不会注入：

- 叙事视角：第一人称、第三人称有限、第三人称全知、多视角。
- 基调氛围：例如热血、冷峻、悬疑、轻松、压抑等。
- 禁止事项：例如禁用桥段、禁用语气、禁用现代词、禁止突然换 POV。
- 写作风格约束：例如句长、对白比例、描写密度、节奏偏好。
- 引用手法：从参考文本中整理出的具体叙事、节奏、结构方法。
- 大师洞察：跨作品提炼出的更高层写作方法论。
- 写作卡片：从参考文本中提炼的可迁移写作技巧。

参考文本默认只保存在本地。只有用户主动点击生成写作卡片、引用手法或大师洞察时，才会把参考文本发送给 AI。

## 🧐 审稿与修改

审稿分为两类：

- 🔎 **一致性检查**：检查人物、时间线、世界观、伏笔、地点和前后逻辑冲突。
- 💡 **专家意见修改**：从节奏、冲突、人物动机、转折、爽点、文风等角度提出优化建议。

手动审稿时，用户可以浏览建议并勾选需要修改的条目；一键生成时也可以开启审稿流程。专家意见默认不会强行覆盖正文，除非用户选择应用。

## 🔐 数据与隐私

- 💾 项目数据保存在浏览器本地 IndexedDB。
- 🔑 API Key 存在本地浏览器数据中，请不要把带密钥的浏览器数据或备份上传到公开位置。
- 📚 参考文本不会自动发送给 AI。
- 🧳 JSON 项目备份会包含项目设定、大纲、章节、创作规则、写作卡片、审稿报告和长期记忆。
- 📝 TXT / Markdown / HTML 正文导出不会包含参考文本和写作卡片。

## 📱 移动端说明

V4.3.8-exG-mobile 在手机窄屏下会自动切换成独立 App 式界面：

- 顶部 App Bar + 底部 Tab Bar。
- 写作页以正文编辑器为核心。
- 章节切换、生成、续写、去 AI 等常用动作放到底部工具区。
- 关键点、批注、记忆、前章回顾、审稿建议通过底部抽屉查看。
- 设置、一键生成、审稿、历史等长内容弹窗改为移动端 Sheet。

桌面端仍保持 V4.3.7 的原有布局和使用方式。

## 💡 使用建议

- 📋 模型较便宜或上下文较短时，优先用摘要模式。
- 📚 使用长上下文云端模型时，可以尝试全文模式。
- 🧠 写几十章以上时，建议开启章节摘要卡、长期记忆和章节承接。
- 🧹 小模型 AI 味较重时，可以把去 AI / 改写任务路由到更擅长文字润色的模型。
- 💾 重要项目请定期导出 JSON 备份。
